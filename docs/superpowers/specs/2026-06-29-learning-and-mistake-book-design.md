# 单词学习与错题本闭环系统设计说明书

**文档状态**: 已批准  
**日期**: 2026-06-29  
**作者**: Antigravity  

---

## 1. 业务目标与核心需求
为了让学员在项目中的“背词/学习课程”体验更加紧凑、流畅且具正向激励，本设计通过整合多模态语音朗读、平滑自动跳转机制，以及双端呼应的“专项闯关消杀”错题本，构建从**输入 -> 纠错 -> 复习 -> 销账**的完整闭环。

---

## 2. 核心交互与设计规范

### 2.1 紧凑型单词学习页 (`Course/Learn/index.vue`)
1. **自带语音朗读 (Auto Pronunciation)**：
   - 当组件加载或 `currentIndex` 发生切换时，系统自动调用 `playAudio(currentWord.word)` 进行高清英文发音朗读。
2. **正确后平滑跳转 (Smooth Auto-Advance)**：
   - 监听每个字母输入完成后的状态校验；当所有字符拼写正确 (`wordList.every(i => i.isTrue)`) 时，触发成功反馈；
   - 输入框呈现绿色发光边框，自动朗读一次单词发音作为记忆强化；
   - **延迟 600ms** 后自动执行 `currentIndex++`，平滑过渡到下一个单词，无需手动点击鼠标。
3. **卡壳收录与快捷键 (Quick Mistake Capture)**：
   - 在拼写输入框下方新增操作栏：**“💡 不会拼？一键显示答案并记入错题”**（支持按 `Tab` 键一键触发）；
   - 点击/按键后，自动将正确字母填充入框，同时调用后端 API 收录进错题本，页面右上角悬浮胶囊计数值即时 +1。

### 2.2 双端入口与闯关消杀错题本
1. **双端呼应的入口布局**：
   - **课程背词页右上角**：常驻悬浮胶囊按钮 `📖 错题攻克 (N)`，点击即可就地展开错题消杀弹窗；
   - **课程大厅 (`Course/index.vue`)**：在“精选课程 / 我的课程”旁新增第三个标签页 **“📖 专属错题本”**，方便用户统筹复习。
2. **专项闯关消杀模式 (Mistake Challenge Mode)**：
   - 告别枯燥的传统列表，错题以**沉浸式抽认拼写卡**单张呈现；
   - 用户听音拼写，每拼对一词，系统自动播放成功提示音，调用销账 API 将该词彻底移出错题库；
   - 界面计数器实时倒数（例如“剩余待攻克：5 -> 4 -> 3”），全量消杀完毕后展示烟花/庆祝空状态提示，给予强烈成就感。

## 3. 系统架构与后端数据链路

### 3.1 数据模型定义 (Prisma Schema 映射)
复用系统现有的 `WordBookRecord` 关联表，通过 `isMaster` 布尔字段做状态标记，无需执行繁重的底层 Schema 迁移：
- `isMaster: false`：表明用户曾拼错或主动收录，处于“错题未掌握”状态。
- `isMaster: true`：表明用户已掌握或通过了错题关卡消杀。

### 3.2 接口规范定义 (`server` 模块)
后端 NestJS 新增以下路由与业务逻辑处理：

| 请求方式 | 路由接口 | 参数说明 | 业务职责 |
| :--- | :--- | :--- | :--- |
| **POST** | `/api/v1/learn/word/mistake` | `{ wordId: string }` | 记录错题：使用 `upsert`，若无记录则新建 `isMaster: false`，若有则更新为 `false`。 |
| **GET** | `/api/v1/learn/word/mistakes` | 无 (从 JWT 取 `userId`) | 获取列表：查询该用户所有 `isMaster: false` 的单词记录及关联单词详情。 |
| **POST** | `/api/v1/learn/word/mistake-resolve` | `{ wordId: string }` | 错题销账：更新 `WordBookRecord` 状态为 `isMaster: true`。 |

---

## 4. 边界条件与异常处理
1. **重复收录处理**：若用户在同一单词上多次点击“加入错题”，后端 `upsert` 操作保证幂等性，不会产生脏数据。
2. **网速慢/音频失败降级**：若音频加载超时，不阻塞前端 600ms 后的自动平滑跳转逻辑。
3. **错题本为空状态**：若用户无错题，错题本页面呈现专属插画与文案“🎉 太棒了！当前没有任何错题，继续保持！”。
